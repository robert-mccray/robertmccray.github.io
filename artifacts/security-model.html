Core Principles

  • Least privilege by default (roles, scopes, tenant boundaries)
  • Defense-in-depth (identity, network, app, data)
  • Auditability (every sensitive action produces a trail)
  • Safe sharing (time-bound, scoped, revocable access)
  • Hardening incrementally (roadmap-driven maturity)

Trust Boundaries

  • Public boundary: browser/mobile → gateway edge
  • Service boundary: gateway → internal APIs
  • Data boundary: services → DB/blob/private endpoints
  • Analytics boundary: curated datasets → governed warehouse
  • Admin boundary: privileged actions → strict RBAC + logging

Threat Model

A focused set of realistic threats for data platforms handling sensitive information.

Identity & Session Threats

  • Credential theft / token replay
  • Privilege escalation via weak roles
  • Over-permissioned service accounts
  • Broken access control (tenant data leakage)
RBAC JWT scopes Least privilege

Data Exfiltration

  • Public exposure of storage buckets/blobs
  • Excessive query access to PHI/PII tables
  • Insecure sharing links without scope/expiry
  • Analytics warehouse access without governance
Private endpoints Encryption Governed access

Abuse & Availability

  • API abuse, brute force, scraping
  • Upload storms (expensive file processing)
  • Queue backlog / job starvation
  • DoS on gateway or hot endpoints
Rate limits WAF rules Async jobs

Supply Chain & Misconfig

  • Secrets in code / env leakage
  • Insecure container images
  • Misconfigured network/security groups
  • Weak pipeline permissions
Key Vault CI checks IaC guardrails

Security Controls

Controls mapped to platform layers. These are the “proof artifacts” a reviewer expects.

Identity & Access

  • JWT auth + role-based access control (RBAC)
  • Tenant-aware authorization (no cross-tenant reads)
  • Admin actions gated behind stronger roles
  • Short-lived tokens + secure refresh strategy
OAuth/JWT RBAC Tenant scope

Network & Perimeter

  • WAF + gateway policy enforcement
  • Private endpoints for DB/storage where applicable
  • Segmented subnets (public ingress vs private services)
  • Ingress allow-lists for admin/service traffic
WAF VNET/VPC Private access

Data & Storage Protection

  • Encryption at rest + TLS in transit
  • Scoped, time-bound secure share links
  • Metadata separation: file pointers vs PHI fields
  • Analytics access restricted to curated datasets
Encryption Time-bound sharing Governance

Platform Guardrails

  • IaC templates enforce secure defaults
  • Policy-as-code checks in CI
  • Secrets in Vault, not environment files
  • Container image scanning + dependency review
Terraform Policy-as-code CI/CD

Audit Logging & Observability

Audit trails are the difference between “secure” and “provably secure.”

What gets audited

  • Authentication events (login, token refresh)
  • Access to PHI/PII resources (read/write/download)
  • Share link creation, access, and revocation
  • Admin actions (role changes, policy changes)

Audit log schema

A consistent schema makes alerting and investigations possible.

  • actor_id, actor_role, tenant_id
  • action, resource_type, resource_id
  • ip, user_agent, request_id
  • timestamp, result (success/fail)

Detection signals

  • Unusual download volume by a user/tenant
  • Repeated authorization failures
  • Access from unexpected locations
  • Spike in share-link creations
