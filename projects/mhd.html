MHD Health System

A security-by-design platform for PHI/PII ownership, controlled sharing, and analytics readiness — built with multi-tenant boundaries, audit-friendly access patterns, and production-first architecture.

Problem

People lose access to medical/performance history when they change providers, teams, or systems. MHD focuses on making history portable, shareable, and governed — without compromising privacy.

Core Requirements

  • Secure uploads + structured medical history
  • Multi-tenant access boundaries + RBAC
  • Time-scoped, auditable secure sharing
  • Async ingestion (NLP/extraction) without blocking APIs
  • Analytics handoff to governed warehouse access
PHI/PII RBAC Audit

Constraints

  • PHI/PII handling expectations (audit, least privilege)
  • Performance under upload & extraction workloads
  • Clear security boundaries (identity, network, data)
  • Repeatable deployments + guardrails
Latency Isolation Repeatability

What I Built

This system was built to prove end-to-end platform thinking: secure ingestion, tenant boundaries, controlled sharing, and a clear path to production hardening.

Secure Upload + Structured History

Users can upload medical records and track injury history/forms in a structured way. Upload flows are designed for PHI safety, with metadata capture and clear ownership boundaries.

Uploads Metadata Ownership

Controlled Sharing (Audit-Friendly)

Sharing is treated as a governed workflow: time-scoped access, least-privilege permissions, and visibility into who accessed what (audit posture).

Expiry Scoped Access Audit

Async Processing Path

Heavy tasks (extraction/NLP) are designed to move out of the request path so the API stays responsive. This is the pattern that scales cleanly into queues + workers in production.

Jobs/Queues Non-blocking Scalable

Security Controls

Security is treated as part of the architecture — not a bolt-on.

Identity & Access

  • JWT auth, RBAC, scoped permissions
  • Least privilege by tenant + role
  • Auditable access patterns (who/what/when)

Production Roadmap

What moves this from a strong system into an enterprise-grade platform.

Reliability

  • Async job orchestration + retries + dead-letter queues
  • Health probes + autoscaling policies
  • Rate limiting & request shaping at gateway

Compliance Depth

  • Formal audit log schema & export
  • Policy-as-code guardrails in CI/CD
  • Key management hardening + rotation

Analytics Readiness

  • Curated datasets pushed to Snowflake with governance
  • Row-level controls & access controls by tenant
  • Monitoring for pipeline freshness + lineage
