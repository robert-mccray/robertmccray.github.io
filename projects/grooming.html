CASE STUDY — AZURE DATA PLATFORM

Living Care Grooming Platform

A role-based booking + approval workflow system for grooming services in living care facilities — designed with audit logging, data governance, and analytics readiness from day one.

Problem

Grooming services inside living care facilities often run on informal scheduling, manual tracking, and inconsistent documentation — leading to missed appointments, limited operational visibility, and weak audit trails for exceptions and overrides.

Core Requirements

  • Role-based booking and approvals (vendor ≠ staff)
  • Reliable schedule tracking with explicit state transitions
  • Audit logging for changes, overrides, and access patterns
  • Governed analytics path for reporting (lakehouse-lite)
  • Secure sharing for facility leadership and read-only viewers
RBAC Audit Analytics

Constraints

  • Minimal-cost dev environment (scale-to-zero patterns)
  • Strict access boundaries across multiple roles
  • Legacy datasets (CSV/Excel) requiring migration + validation
  • Analytics without expensive always-on warehousing
Cost Governance Migration

What I Built

This project was intentionally designed to demonstrate end-to-end Azure data platform capability: operational workflows (OLTP), data governance, ETL pipelines, and BI visibility — with cost-aware architecture.

Approval-Based Booking Workflow

Booking is treated as a governed workflow: vendors request slots, facilities approve/deny, and staff can override with reason codes. State changes are captured for auditability.

Workflow State Machine Exceptions

Governance + Secure Sharing

Identity-first access (Entra roles), least-privilege service access (managed identity), and audit events as first-class records. Reporting access is governed through dataset permissions.

Entra ID Least Privilege Audit

Lakehouse-Lite Analytics

Operational + audit data exported through ADF into ADLS using Bronze/Silver/Gold layers. Silver enforces data contracts; Gold publishes star-schema datasets for BI consumption.

Core services: Azure SQL (OLTP + audit), Data Factory (pipelines), ADLS Gen2 (Bronze/Silver/Gold), serverless SQL views (Silver + Gold), Power BI dashboards, Azure Monitor / Log Analytics for observability.

Key architecture decision: deliver enterprise-grade governance and analytics readiness with serverless / scale-to-zero patterns, avoiding always-on warehouses during early validation.

Data Quality Contracts (Silver)

  • Reject reason codes normalized (UPPER)
  • Facility code required (defaults to UNKNOWN when missing)
  • Typed timestamps for consistent time-series reporting

Talking point: “Rejects are first-class entities, not discarded errors.”

Results

Practical outcomes and platform signals: governance posture, operational visibility, and cost-aware design.

Operational Visibility

  • Facility-level utilization metrics (bookings, cancellations, completion rate)
  • Vendor throughput and schedule adherence insights
  • Exception tracking (overrides, denials, no-shows)

Governance & Auditability

  • Append-only audit trail for critical actions
  • Clear separation of operational vs analytics access
  • Quality enforcement through Silver contracts + reject facts

Performance & Cost Discipline

  • Scale-to-zero dev patterns where applicable
  • Curated lake layers instead of always-on warehousing
  • Partitioned outputs to reduce scan cost and improve responsiveness
